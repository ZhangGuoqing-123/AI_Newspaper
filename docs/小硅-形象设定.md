# 小硅 · 硅谷速递品牌吉祥物设定

> 定稿日期：2026-06-03。本文件锁定 IP 形象的概念、配色与生成参数，方便以后复现/迭代时对得上。

## 一、概念

**小硅** = 一个「屏幕脸」的 AI 速递小机器人。

- **命名逻辑**：硅 = 芯片 = 机器人；速递 = 它怀里抱着一份小报纸，寓意"把新闻递给你"。
- **风格基调**：光滑现代的科技感、圆润亲和，**干净高级、不幼态**（刻意避开奶萌大头娃方向）。
- **为什么不是拟人脸**：试过 3D 半写实 / 写实真人 / 二次元立绘，都要么突兀、要么幼态、要么蹦出"奶奶/银发"，且即梦通用模型对真人脸有强照片惯性。改用非人吉祥物后一次到位。

## 二、配色（与 App 主题一致）

取自前端 `src/index.css`：

| 用途 | 值 | 说明 |
|---|---|---|
| 主色 | `hsl(217 91% 60%)` | 明亮科技蓝 |
| 渐变终点 | `hsl(238 84% 67%)` | 靛紫 |
| 品牌渐变 | `linear-gradient(135deg, hsl(217 91% 60%), hsl(238 84% 67%))` | App 同款 |

小硅主体走"科技蓝 → 靛紫"渐变，与 App 一脉相承。

## 四、为什么是「屏幕脸」——顺手解决口播视频

小硅的脸是一块屏幕（显示眼睛+嘴）。做数字人口播视频时：

- **不用 SadTalker 对真人脸做对口型**（机器人本就不是真人脸，SadTalker 也驱动不了）；
- 直接让**屏幕上的嘴 / 声波**随音频波形动即可——**成本几乎为零、对口型 100% 准、且完全契合"它是机器人"的设定**。
- 面试讲法：把 IP 设计成屏幕脸，是一次"用可控动画替代不可控生成式、顺带压成本"的取舍。呼应讲法稿《项目深挖讲法》3.3「能可控就别用生成式」。
